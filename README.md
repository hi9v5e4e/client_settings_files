# 1. Смысл репозитория

Данный репозиторий предназначен для хранения резервных копий настроек (бэкапов) различных VPN-клиентов. Файлы позволяют быстро восстановить или перенести настройки приложения на другое устройство.

В настоящее время репозиторий содержит конфигурации для мобильных устройств под управлением Android и таких клиентов, как **[Exclave](https://github.com/ExclaveNetwork/Exclave)** и **[v2rayNG](https://github.com/2dust/v2rayNG)**.

В дальнейшем планируется добавление новых клиентов, платформ и конфигураций по мере развития проекта.

# 4. Как использовать файлы

## В Exclave
Чтобы добавить файл восстановления в этот клиент, нужно:
1. Скачать файл `exclave_mobile_23.06.2026.14.16.1.json`
2. Зайти в приложение
3. Найти в меню вкладку `Инструменты`
4. Зайти в раздел `Бэкап`
5. Нажать на `Импорт из файла`
6. Найти и выбрать скачанный файл с нужным названием
7. В окне `Импорт` не убирать никаких галочек, нажать `ОК`

## В v2rayNG
Чтобы добавить файл восстановления в этот клиент, нужно:
1. Скачать файл `v2rayng_mobile_23.06.2026.14.19.1.zip`
2. Зайти в приложение
3. Найти в меню вкладку `Резервное копирование`
4. Нажать на `Восстановление конфигурации`
5. Выбрать `Локально`
6. Найти и выбрать скачанный файл с нужным названием

> Все конфигурации подстроены "под себя", то есть я оставил так, как пользуюсь я.

P.S. В этих конфигурациях я нашёл группы политик (балансировщики), которые автоматически выбирают сервер с быстрым откликом или скоростью (а в Exclave ещё можно настроить, через какое время проводить проверки отклика серверов). Такая популярная настройка есть в Karing и Hiddify например, но почему-то у меня именно в них эта функция работает плохо
